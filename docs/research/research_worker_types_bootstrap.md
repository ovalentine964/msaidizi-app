## 2. Bootstrap Experience Design

### First-Launch Conversation Flow (Voice-First)

```
Msaidizi: "Habari! Mimi ni Msaidizi — msaidizi wako wa biashara. 
           Jina lako nani? (Hello! I'm Msaidizi — your business helper. 
           What's your name?)"

Worker: "Mimi ni Mama Amina"

Msaidizi: "Poa Mama Amina! Sasa, biashara yako ni ipi? 
           (Great Mama Amina! What's your business?)"
           
[Auto-detect from speech patterns + offer options via voice]
"Je, wauza mboga? Au unafanya kazi nyingine?"
(Do you sell vegetables? Or do you do other work?)

Worker: "Nauza mboga soko" (I sell vegetables at market)

Msaidizi: "Sawa! Soko gani? (Which market?)"

Worker: "Gikomba"

Msaidizi: "Poa! Sasa nikuulize maswali machache kuhusu biashara yako..."
(Great! Now let me ask a few questions about your business...)

[Quick voice-based questionnaire:]
1. What products do you sell most? (learns product vocabulary)
2. What are your usual prices? (learns price ranges)
3. What time do you open/close? (learns schedule for alerts)
4. Do you use M-Pesa? Cash? Both? (learns payment methods)
5. Do you have a phone number for WhatsApp reports? (optional)

Msaidizi: "Sawa Mama Amina! Msaidizi wako tayari kusaidia. 
           Anza kuniambia mauzo yako leo!"
(Alright Mama Amina! Your Msaidizi is ready to help. 
 Start telling me about your sales today!)
```

### What Bootstrap Learns (Personalization Data)

| Data Point | How It's Learned | What It Enables |
|-----------|-----------------|-----------------|
| Worker's name | Asked directly | Personalized greetings |
| Business type | Asked + auto-detected from products | Relevant CFO advice, industry benchmarks |
| Location/market | Asked | Local market intelligence, pricing data |
| Products sold | Listed during bootstrap | STT vocabulary pre-loaded, faster recognition |
| Price ranges | Asked for top products | Anomaly detection (flag unusual amounts) |
| Operating hours | Asked | Smart alert timing (briefings at right time) |
| Payment methods | Asked | Transaction parsing (M-Pesa SMS vs voice) |
| Language preference | Auto-detected from speech | UI language, voice language |
| WhatsApp (optional) | Asked | Report delivery via WhatsApp |

### Bootstrap Data Storage

```kotlin
data class WorkerProfile(
    val name: String,                    // "Mama Amina"
    val businessType: BusinessType,      // VEGETABLE_VENDOR
    val market: String,                  // "Gikomba"
    val location: Location,              // Nairobi, Kenya
    val products: List<Product>,         // [{name: "tomato", price: 10}, ...]
    val operatingHours: OperatingHours,  // 6AM - 6PM
    val paymentMethods: List<PaymentMethod>, // [MPESA, CASH]
    val language: Language,              // SWAHILI
    val dialect: Dialect,                // STANDARD_SWAHILI
    val whatsappPhone: String?,          // "+254712345678"
    val bootstrapDate: Instant,          // When they first joined
    val msaidiziName: String,            // What they named their AI ("Msaidizi")
)
```

### Naming the Msaidizi (Endowment Effect)

Following Kahneman's endowment effect (from the review):
- Workers can name their Msaidizi whatever they want
- Creates emotional attachment ("This is MY Msaidizi")
- Default suggestion: "Msaidizi" (helper)
- Worker might name it: "Biashara" (business), "Rafiki" (friend), "Mama" etc.

## 4. Worker Type Auto-Detection

### How Msaidizi Detects Worker Type from Speech

```kotlin
// During bootstrap and first transactions, detect type from:
val detectionSignals = mapOf(
    "VEGETABLE_VENDOR" to listOf("mboga", "nyanya", "vitunguu", "soko", "kilo"),
    "BODA_BODA" to listOf("abiria", "route", "petroli", "fare", "nduthi"),
    "DUKAWALLAH" to listOf("duka", "stock", "customer", "deni", "receipt"),
    "MAMA_LISHE" to listOf("chapati", "ugali", "nyama", "mboga", "chakula"),
    "FUNDI" to listOf("repair", "part", "kazi", "client", "bei"),
    "M_FUGAJI" to listOf("ng'ombe", "kuku", "malisho", "maziwa", "nyama"),
    "MJENGO" to listOf("matofali", "siment", "kazi", "mjengo", "mason"),
    "MPESA_AGENT" to listOf("mpesa", "transaction", "commission", "cash", "float")
)

// Confidence scoring: if 3+ signals match → auto-detect type
// If ambiguous → ask: "Biashara yako ni ya aina gani haswa?"
// (What type of business do you do exactly?)
```
